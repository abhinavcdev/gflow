Configuration

Configure gflow with a .gflow.yml file in your repository root.

Full Example

# .gflow.yml
version: "0.1.0"

provider:
  name: github
  token_env: GITHUB_TOKEN
  owner: your-org
  repo: your-repo

branching:
  main: main
  develop: develop
  use_develop: false
  prefixes:
    feature: feature/
    bugfix: bugfix/
    hotfix: hotfix/
    release: release/

pr:
  default_base: main
  auto_assign: true
  draft: false
  merge_method: squash
  delete_branch_on_merge: true

commit:
  convention: conventional
  require_scope: false
  types:
    - feat
    - fix
    - docs
    - refactor
    - test
    - chore

Provider

Configure your Git hosting provider (currently GitHub only).

  • name — Provider name (github)
  • token_env — Environment variable for auth token
  • owner — Repository owner or organization
  • repo — Repository name

Branching

Define your branching strategy.

  • main — Main/production branch
  • develop — Development branch (optional)
  • use_develop — Use git-flow style with develop branch
  • prefixes — Branch name prefixes for each type

Pull Requests

Configure PR creation and merging behavior.

  • default_base — Default target branch for PRs
  • auto_assign — Auto-assign PR creator
  • draft — Create PRs as drafts by default
  • merge_method — merge, squash, or rebase
  • delete_branch_on_merge — Auto-delete merged branches

Commit Conventions

Enforce commit message standards.

  • convention — conventional, angular, or none
  • require_scope — Require scope in commit messages
  • types — Allowed commit types

Hooks

Run commands at specific lifecycle points.

hooks:
  pre_pr:
    - "go vet ./..."
    - "go test ./..."
  post_merge:
    - "make deploy"
