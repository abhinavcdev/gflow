Getting Started

Get up and running with gflow in under 5 minutes.

Installation

Install gflow using Go:

go install github.com/abhinavcdev/gflow@latest

Or download the latest binary from GitHub Releases.

Verify the installation:

gflow version

GitHub Token Setup

gflow needs a GitHub personal access token to create PRs and manage branches.

Generate a token

  1. Go to GitHub Settings → Developer settings → Personal access tokens
  2. Click Generate new token (classic)
  3. Give it a descriptive name like "gflow CLI"
  4. Select scopes:
    • repo (full control of private repositories)
    • admin:repo_hook (if using branch protection)
  5. Click Generate token
  6. Copy the token (you won't see it again!)

Set the token

Add the token to your shell profile (~/.zshrc or ~/.bashrc):

export GITHUB_TOKEN="your_token_here"

Then reload your shell:

source ~/.zshrc

Initialize gflow

In your repository, run:

gflow init

This will:

  • Auto-detect your GitHub repository
  • Create a .gflow.yml config file
  • Set up branching conventions
  • Configure PR and commit rules

The interactive setup will guide you through each step. You can accept the defaults or customize to your team's workflow.

Your First Workflow

Let's create a feature branch, make a PR, and merge it.

1. Start a feature branch

gflow start feature user-authentication

This creates feature/user-authentication from main and switches to it.

2. Make your changes

Edit files, commit as usual:

git add .
git commit -m "feat: add user login endpoint"

3. Create a pull request

gflow pr "Add user authentication"

This will:

  • Commit any uncommitted changes
  • Push your branch to origin
  • Create a PR on GitHub
  • Show you the PR URL

4. Check your status

gflow dash

See your branch status, PR details, and CI check results in real-time.

5. Merge and clean up

Once CI passes and you have approvals:

gflow finish

This will:

  • Verify all CI checks passed
  • Merge the PR (squash by default)
  • Switch back to main
  • Pull the latest changes
  • Delete the feature branch locally and remotely
